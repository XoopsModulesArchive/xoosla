Installing Xoosla

First time installation

Preface:

Xoosla CMS is an open-source Object-Oriented Web publishing system written in PHP. It is an ideal tool for developing small to large dynamic community websites, intra company portals, corporate portals, weblogs and much more.

Xoosla CMS has been released under the terms of the GNU General Public License (GPL) and is free to use and modify as long as you retain the copyright notices. It is free to redistribute as long as you abide by the distribution terms of the GPL.

Requirements

  • WWW Server (Apache, IIS, Roxen, etc)
  • PHP 5.2.x or higher
  • MySQL 5.1 or higher

Before You Install

  1. Setup WWW server, PHP and database server properly.
  2. Prepare a database for your Xoosla CMS site.
  3. Prepare user account and grant the user the access to the database.
  4. Make the directory /uploads/ and the file mainfile.php writable
  5. Turn cookie and JavaScript of your browser on.

Installing locally

If you are running a local environment for development or testing, make sure that you have the previous requirements met. Once this has been done, copy the contents of the htdocs directory (from the Xoosla CMS distribution file or SVN) to the root document path of your web environment. Once the files are copied there, you can start the install by typing "http://yoursite.com". This will start the install process.

Installing on a hosted platform

If you are running in a hosted environment, Either, unpack Xoosla archive to your local hard drive and then upload to the document root or a sub-directory on your webhosting, or upload the Xoosla archive to the document root or a sub-directory on your webhosting and unpack the archive either with tellnet or SHH.

Once the files have been upload and are ready to install, please point your browser to the document root or the sub-directory where you will be prompted to start the Xoosls installation process.

Continuing The Install

The installer has six install steps, follow each step as requested till you reach the end. If you meet any errors, make the required correction and then go ahead to the next step. If you meet any errors that you cannot fix, please jump over to our forum and ask for some help
